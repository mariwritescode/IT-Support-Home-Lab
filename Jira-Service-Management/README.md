# Jira Service Management

## Overview

Jira Service Management was used to simulate a Help Desk ticketing environment for the IT Support Home Lab.

The platform was used to document incidents, track troubleshooting efforts, identify root causes, record resolutions, and verify successful remediation.

## Ticket Workflow

Each incident followed a standard support process:

1. Ticket Creation
2. Investigation
3. Root Cause Identification
4. Resolution
5. Verification
6. Ticket Closure

## Documented Incidents

### RPS-1 / HD-001

**User Unable to Access Accounting Share**

* Investigated user permissions
* Identified NTFS permission issue
* Restored access to the Accounting share

### RPS-2 / HD-002

**Accounting Network Drive Missing**

* Investigated missing mapped drive
* Restored network drive mapping
* Verified access to shared resources

### RPS-3 / HD-003

**Workstation Unable to Resolve Domain Resources**

* Investigated DNS configuration
* Corrected DNS settings
* Verified successful domain name resolution
